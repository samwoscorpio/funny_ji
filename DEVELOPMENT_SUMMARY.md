# 好玩的 Ji 开发进展总结

更新时间：2026-07-10

## 本次地图式战术原型

本次审计确认：旧战斗内核包含 19 名英雄、14 个基础动作（Ji、3 种防御、10 种攻击）以及各英雄主动技能；单挑结算入口为 `resolveRound`，混战结算入口为 `resolveMeleeRound`，重开入口为 `resetGame`，在线房间入口由 `server.py` 的 `/api/rooms` 系列接口提供。

已新增 `tactical.js` 与新的主界面骨架：7×7 固定地图、每回合 1 格路径规划、行动卡选择、目标格预判、AI BFS 移动、同时移动冲突、地图能量点、中央据点和 4 分胜利均已接入。旧 `BALANCE`、`HEROES`、历史记录结构、混战代码和房间代码未删除；地图模式只将它们作为兼容的规则内核调用。

已验证：经典武者、刺客、铁壁、牧师、狼人、占星家、药师、圣骑士、忍者、木偶，以及已有的多人伤害与单点防御测试均通过。在线房间的地图路径同步仍未实现，因此当前仅将其保留为旧版能力，主界面明确标注为开发中。

## 项目概况

《好玩的 Ji》是一个从童年回合制拍手游戏扩展出来的网页对战游戏。当前版本以浏览器端为主，已经具备人机单挑、本地房间 1v1、混战测试、英雄机制、头像、说明书、历史对局、基础动画和 GitHub Pages / Render 部署基础。

项目主要文件：

- `index.html`：页面结构、弹窗入口、操作面板、英雄状态区。
- `styles.css`：整体视觉、移动端布局、英雄卡片、混战界面、动画样式。
- `game.js`：核心规则、英雄定义、AI、UI 渲染、混战逻辑、历史记录。
- `server.py`：本地 / 云端房间 1v1 的轻量服务端。
- `match-log.md`：对战记录数据结构说明和人工备份模板。
- `.github/workflows/deploy-pages.yml`：GitHub Pages 静态站自动部署流程。

## 已完成的核心规则

- 基础属性：玩家拥有 HP 和 XP，初始值可由英雄配置决定。
- 基础动作：
  - `Ji`：获得 XP。
  - 防御：小防、中防、大防，并支持更高等级的超防 / 无敌。
  - 攻击：1 到 10 费攻击，其中 1 到 5 费已有命名：小刀、大刀、冰刀、火刀、鬼刀。
- 平衡配置集中在 `BALANCE` 对象中，费用、伤害、防御、英雄数值可以统一调整。
- HP 上限机制：多数英雄上限为初始 HP + 2，部分英雄通过自身 `maxHp` 体现。
- 对攻判定：双方互相攻击时，会比较攻击强度，弱者受伤，强者不受该来源伤害。
- 防御机制已扩展为：
  - 全局防御：主动出防、无敌、部分技能防御。
  - 单点防御：攻击某目标时获得针对该目标的防御；部分指向技能也会附带单点防御。
- 多来源伤害结算：同一回合受到多个有效攻击时，实际扣血取最大伤害，不简单叠加。
- 攻击来源保留：例如舞女转移伤害时，伤害来源仍然是原攻击者。

## 已完成的英雄

当前已经实现的英雄包括：

- 经典武者：无额外机制。
- 牧师：3 HP；只能使用 1/5/10 费攻击；拥有庇护和疗愈。
- 刺客：1 HP；Ji 获得 4 XP。
- 吸血鬼：2 HP；造成伤害后回复 HP。
- 虚荣勇士：2 HP；开局自带 2 XP。
- 狼人：2 HP；首次低于 2 HP 后狂暴，获得两回合无敌，之后 Ji +2，伤害翻倍，并切换狂暴头像。
- 冰法：2 HP；拥有寒冰碎片、冰刀折扣、Ji 刀、暴击回血机制；第一回合不能出 Ji 刀，最多连续两次。
- 占星家：2 HP；鬼刀可吸血；拥有预判和汲取。
- 破阵手：4 HP；偏攻击增强。
- 铁臂：4 HP；防御手势防御值 +1。
- 聚气师：2 HP；连续 Ji 加成；造成伤害后进入结束阶段，可选择追索。
- 平衡 bot：2 HP；Ji 获得 2 XP。
- 舞女：Ji 获得 2 XP；击中目标后强制目标下回合 Ji；拥有小转 / 大转，可转移一定强度以下的攻击。
- wood：高血量测试靶，用于调试和测试。

## UI 与交互进展

- 游戏标题已改为“好玩的 Ji”。
- HP 使用红色爱心显示，不再显示血量上限。
- 英雄状态栏加入头像；已有头像资源包括刺客、牧师、虚荣勇士、吸血鬼、狼人、占星家、冰法、舞女等。
- 点击英雄头像可以查看英雄技能详情。
- 顶部增加“说明书”入口，说明基础规则、动作、阶段、防御类型和混战。
- 顶部回合计数器旁边增加“历史对局”入口。
- 移动端布局已多轮优化：
  - 单挑中英雄框与出招框更紧凑。
  - 防御按钮在窄窗口下可以保持同排。
  - 混战中玩家卡片、敌人卡片、出招入口减少重复信息。
- 混战交互已改为：
  - Ji 和小防放在当前玩家卡片内。
  - 点击敌方卡片后，在目标卡片内选择攻击或指向技能。
  - 攻击可以分配给多个目标。
  - 攻击和指向技能不能同回合混用。
  - 提交 / 清空按钮移动到玩家头像旁边，避免撑大卡片。
  - 按 Enter 可以提交当前混战选择。
- 死亡效果：英雄头像上增加灰色遮罩和 `OUT` 标记。

## 对战模式进展

### 人机单挑

已经可玩。AI 已做基础智能优化，例如：

- 玩家 XP 为 0 时，电脑减少无意义防御。
- 玩家 XP 不足以打穿中防 / 大防时，电脑减少过度防御。
- AI 会根据费用、防御、攻击可用性做简单选择。

### 房间 1v1

已经有本地房间代码模式和 `server.py`：

- 可以创建房间。
- 可以通过房间代码加入。
- GitHub Pages 只能托管静态页面，不能运行房间服务器。
- Render / Docker 平台可以运行 `server.py`，用于真正的在线 1v1。

### 混战测试

当前实现的是本地测试版本：

- 1 个真人 + 2 个电脑。
- 可以分别给电脑 A 和电脑 B 选择英雄。
- 支持多目标攻击分配。
- 支持指向技能框架。
- 支持单点防御、多来源伤害、转移伤害、强制 Ji 等核心混战规则。

完整的线上多人混战尚未实现。

## 对战记录系统

当前已完成：

- 玩家昵称输入。
- 对局结束后记录当前用户历史。
- 记录字段包括玩家昵称、模式、英雄、对手英雄、胜负、回合数、结束时间。
- 历史记录保存在浏览器 `localStorage`，键名为 `clapDuel.matchHistory.v1`。
- `match-log.md` 记录了日志字段说明和人工备份模板。

当前限制：

- 记录是本机浏览器级别，不是服务器级别。
- 换浏览器、清缓存、换设备后不会自动同步。
- 目前还没有真正的账号系统或云端日志数据库。

## 动画进展

已经加入基础像素街机风格动画方向：

- 攻击 / 技能会有更具指向性的动画表现。
- 头像死亡蒙版显示 `OUT`。
- 当前动画主要是基础攻防和目标反馈。

后续仍需补全：

- 每个英雄的专属技能动画。

## 当前主要问题

### 1. 规则复杂度开始变高

英雄机制已经包括治疗、吸血、暴击、碎片、预判、汲取、强制 Ji、转移伤害、结束阶段选择等。当前大多数逻辑集中在 `game.js`，继续增加英雄后，维护成本会明显上升。

建议后续拆分：

- `balance.js`：数值配置。
- `heroes.js`：英雄定义。
- `rules.js`：通用结算。
- `ui.js`：渲染和交互。
- `server.py` 或后续服务端：在线房间状态。

### 2. 混战仍是本地测试版

当前混战支持 1 真人 + 2 AI，但还不是完整 N 玩家在线混战。真正上线混战还需要解决：

- 多真人同时选招。
- 回合锁定和超时。
- 断线重连。
- 观战 / 房主控制。
- 服务器权威结算，防止客户端篡改。

### 3. 线上 1v1 与静态版能力不同

GitHub Pages 只能玩静态版本，不能运行 WebSocket / HTTP 房间服务。玩家如果要远程房间对战，需要打开 Render 服务地址，而不是 GitHub Pages 地址。

后续需要在 UI 或 README 中明确区分：

- GitHub Pages：静态试玩。
- Render：在线房间版。

### 4. README 部分部署说明需要更新

`README.md` 里 GitHub Pages 部分仍写着 `Deploy from a branch`，但现在已经新增 GitHub Actions 部署 workflow。建议更新 README，避免以后自己或别人照旧说明配置，重新触发旧的 Pages 自动队列。

### 5. 测试体系还不够正式

目前已有本地测试脚本覆盖了不少核心规则，但还不是标准化工程测试结构。建议后续建立：

- 固定测试目录。
- 每个英雄至少一组核心机制测试。
- 混战结算测试。
- UI 关键流程冒烟测试。
- 部署前自动运行测试。

### 6. 动画系统还处于早期

当前动画能提供反馈，但还没有形成统一的动画事件系统。后续如果每个英雄都有专属动画，建议先定义动画事件类型：

- `attack`
- `block`
- `heal`
- `drain`
- `redirect`
- `crit`
- `death`
- `status`

规则结算只发事件，UI 层根据事件播放动画，这样更容易扩展。

### 7. 历史对局还不是云端记录

当前历史对局只存在本地 `localStorage`。如果以后要支持不同玩家昵称、排行榜、长期战绩，需要服务端数据库或至少一个轻量 JSON 存储方案。

### 8. 部分数值仍需平衡

已经有很多高强度机制，后续需要重点关注：

- 刺客 Ji +4 是否过快。
- 狼人狂暴后两回合无敌 + 伤害翻倍是否过强。
- 冰法碎片暴击回血节奏是否稳定。
- 占星家汲取固定 +4 XP 是否过强。
- 舞女强制 Ji 与伤害转移是否容易形成锁死。
- 牧师治疗 / 无敌是否让多人模式拖太久。

### 9. 英雄说明显示仍需统一

已经做过技能说明优化，但不同英雄的被动、状态、详细说明仍可能在卡片中显示不一致。建议统一成：

- 卡片内只显示短标签。
- 头像详情弹窗显示完整说明。
- 战报显示当回合触发原因。
